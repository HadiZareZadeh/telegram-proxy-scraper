# راهنمای فارسی — جمع‌آوری و تست پروکسی تلگرام

این راهنما برای کسانی نوشته شده که **هیچ تجربه‌ای در برنامه‌نویسی ندارند**. قدم‌به‌قدم توضیح می‌دهد این برنامه چیست، چه کاری می‌کند، و چطور از آن استفاده کنید.

---

## این برنامه چیست؟

**MTProto + V2Ray Scraper** (یا به‌اختصار «جمع‌آور پروکسی») یک نرم‌افزار رایگان است که:

1. **کانال‌ها و گروه‌های تلگرامی** که شما عضو آن‌ها هستید را می‌خواند.
2. **لینک پروکسی** (راه‌های دور زدن فیلترینگ) را از پیام‌ها پیدا می‌کند.
3. **آن‌ها را امتحان می‌کند** تا ببیند کدام‌ها واقعاً کار می‌کنند.
4. **پروکسی‌های سالم** را در یک فهرست ذخیره می‌کند تا بعداً استفاده کنید.

همهٔ این کارها از طریق یک **پنجرهٔ گرافیکی ساده** (دکمه‌ها و متن) انجام می‌شود؛ نیازی نیست دستورات پیچیدهٔ کامپیوتری یاد بگیرید.

---

## چند واژهٔ ساده

| واژه | یعنی چه؟ |
|------|----------|
| **پروکسی** | یک سرور واسط که اینترنت شما را از مسیر دیگری عبور می‌دهد؛ مثل «دروازهٔ دیگر» برای رسیدن به تلگرام یا سایت‌ها. |
| **MTProto** | نوع پروکسی مخصوص **تلگرام**. معمولاً به‌صورت لینک `tg://proxy?...` یا `https://t.me/proxy?...` در کانال‌ها منتشر می‌شود. |
| **V2Ray / Xray** | نوع دیگری از پروکسی/فیلترشکن که با برنامه‌هایی مثل **NekoRay**، **v2rayN** و مشابه استفاده می‌شود. |
| **کانال / گروه منبع** | همان کانال یا گروه تلگرامی که داخلش لینک پروکسی می‌گذارند (مثلاً `@ShadowProxy66`). |
| **اسکرپر (Scraper)** | بخشی از برنامه که پیام‌های تلگرام را می‌خواند و لینک‌ها را جمع می‌کند. |
| **پینگ (Ping)** | «امتحان کردن» پروکسی؛ مثل اینکه ببینید درِ یک خانه باز است یا قفل. |
| **اشتراک (Subscription)** | یک فایل یا لینک که لیست پروکسی‌های سالم را به برنامهٔ فیلترشکن شما می‌دهد تا خودکار به‌روز شود. |

## قبل از شروع چه چیزهایی لازم است؟

### ۱. کامپیوتر

- **ویندوز ۱۰ یا ۱۱** (راهنمای نصب ویندوز در این فایل کامل‌تر است)
- یا **لینوکس** (Ubuntu و مشابه — مراحل کمی متفاوت است)

### ۲. حساب تلگرام

- باید **شمارهٔ موبایل** و **حساب تلگرام** داشته باشید.
- باید **عضو کانال‌ها یا گروه‌هایی** باشید که لینک پروکسی منتشر می‌کنند.

### ۳. کلید API تلگرام (api_id و api_hash)

تلگرام برای برنامه‌های شخص ثالث (مثل این نرم‌افزار) یک **شناسه** و **رمز** می‌دهد. این‌ها مثل «کلید ورود برنامه به تلگرام» هستند؛ **رمز عبور حساب شما نیستند**.

**چطور بگیرید:**

1. در مرورگر بروید به: [https://my.telegram.org/apps](https://my.telegram.org/apps)
2. با **شمارهٔ تلگرام** خود وارد شوید (کد تأیید از اپ تلگرام می‌آید).
3. روی **API development tools** کلیک کنید.
4. اگر اپلیکیشن ندارید، یکی بسازید (نام و توضیح هر چیزی می‌تواند باشد).
5. دو عدد/متن می‌بینید:
   - **App api_id** → مثلاً `26239604`
   - **App api_hash** → یک رشتهٔ طولانی مثل `5d6f1c5f2105bb3a701d0d70d6f4dc46`

این دو را **یادداشت کنید**؛ بعداً در فایل تنظیمات می‌گذاریم.

> **امنیت:** `api_id` و `api_hash` را مثل رمز عبور با کسی به اشتراک نگذارید. اگر لو رفت، از همان صفحهٔ my.telegram.org می‌توانید اپ را حذف و دوباره بسازید.

### ۴. (اختیاری) برنامه‌های دیگر

| برنامه | برای چه؟ |
|--------|----------|
| **Telegram Desktop** | برای باز کردن سریع پروکسی‌های MTProto از داخل این نرم‌افزار |
| **NekoRay** یا مشابه | برای استفاده از پروکسی‌های V2Ray با «اشتراک» |

## نصب — قدم‌به‌قدم (ویندوز)

### مرحله ۱: دانلود پروژه

اگر از **GitHub** یا **فشرده (ZIP)** گرفته‌اید، پوشه را در جایی مثل `D:\project\telegram-proxy-scraper` باز کنید.

داخل پوشه باید فایل‌هایی مثل `setup.cmd`، `app.py` و `README.md` را ببینید.

### مرحله ۲: اجرای نصب‌کننده

1. داخل پوشهٔ پروژه بروید.
2. روی فایل **`setup.cmd`** دوبار کلیک کنید (یا در Command Prompt همان پوشه بنویسید: `setup.cmd`).
3. صبر کنید تا تمام شود. این اسکریپت خودکار:
   - **Python** (زبان برنامه) را اگر نباشد نصب می‌کند (معمولاً با winget)
   - یک محیط مجازی **`.venv`** می‌سازد
   - کتابخانه‌های لازم را نصب می‌کند
   - در صورت نیاز **Xray** را برای تست V2Ray دانلود می‌کند
   - اگر **`config.yaml`** نباشد، از روی نمونهٔ **`config.example.yaml`** می‌سازد

اگر پیام **OK** دیدید، نصب موفق بوده است.

> **نکته:** اولین بار ممکن است چند دقیقه طول بکشد؛ اینترنت باید وصل باشد.

### مرحله ۳: تنظیم فایل `config.yaml`

بعد از نصب، فایل **`config.yaml`** را با **Notepad** (یا هر ویرایشگر متن ساده) باز کنید.

مهم‌ترین بخش‌ها:

```yaml
telegram:
  api_id: 12345678          # ← عدد api_id خودتان
  api_hash: "abcd1234..."   # ← api_hash خودتان (داخل گیومک)
  session_name: mtproto_scraper
  sources:
    - "@نام_کانال_اول"
    - "@نام_کانال_دوم"
  messages_per_source: 500
```

**توضیح ساده:**

| خط | چه بگذارید؟ |
|----|-------------|
| `api_id` | عددی که از my.telegram.org گرفتید |
| `api_hash` | متن طولانی api_hash — **حتماً داخل `"..."` باشد** |
| `sources` | لیست کانال/گروه‌هایی که عضوشان هستید؛ با `@` یا لینک `https://t.me/...` |
| `messages_per_source` | چند پیام آخر هر کانال خوانده شود (۵۰۰ یعنی ۵۰۰ پیام اخیر؛ `null` یعنی همه — خیلی کند) |

**مثال کانال:**

```yaml
  sources:
    - "@ShadowProxy66"
    - "https://t.me/Gp_config"
```

فایل را **ذخیره** کنید (Ctrl+S).

## اجرای برنامه (ویندوز)

بعد از نصب و تنظیم `config.yaml`:

1. دوبار کلیک روی **`app.py`** ممکن است کار نکند؛ روش درست این است:
2. در همان پوشهٔ پروژه، این مسیر را اجرا کنید:

```bat
.venv\Scripts\pythonw.exe app.py
```

یا از **Command Prompt** در پوشهٔ پروژه:

```bat
cd D:\project\telegram-proxy-scraper
.venv\Scripts\pythonw.exe app.py
```

یک **پنجرهٔ «fetch-mtproto control panel»** باز می‌شود.

> **pythonw** یعنی پنجرهٔ سیاه Command Prompt نشان داده نمی‌شود؛ فقط همان پنل گرافیکی.

### لینوکس

```bash
chmod +x setup.sh
./setup.sh
# بعد از ویرایش config.yaml:
.venv/bin/python app.py
```

## آشنایی با پنل کنترل (دکمه‌ها)

| دکمه | کار |
|------|-----|
| **Start Scraper** | به تلگرام وصل می‌شود، کانال‌ها را می‌خواند، لینک‌های جدید را ذخیره می‌کند و **منتظر پست‌های جدید** می‌ماند. |
| **Ping MTProto** | همهٔ پروکسی‌های MTProto ذخیره‌شده را **یک‌بار امتحان** می‌کند؛ سالم‌ها و خراب‌ها جدا می‌شوند. |
| **Ping V2Ray** | همهٔ لینک‌های V2Ray را با **Xray** امتحان می‌کند. |
| **Start Subscription server** | فایل اشتراک را می‌سازد و روی کامپیوتر شما آدرس محلی می‌دهد (برای NekoRay). |
| **Open top … proxies** | سریع‌ترین پروکسی‌های MTProto سالم را در **Telegram Desktop** باز می‌کند. |

### اولین بار: ورود به تلگرام

وقتی **Start Scraper** را می‌زنید:

1. ممکن است در **کادر پایین پنجره** (جایی که لاگ نوشته می‌شود) بخواهد **شمارهٔ موبایل** بدهید — با کد کشور، مثلاً `+98912xxxxxxx`
2. بعد **کد تأیید** از اپ تلگرام روی گوشی
3. اگر **رمز دو مرحله‌ای (2FA)** دارید، همان را هم می‌خواهد

پاسخ‌ها را در **همان کادر ورودی** پایین صفحه تایپ کنید و Enter بزنید.

بعد از ورود، یک فایل نشست در پوشهٔ **`sessions/`** ساخته می‌شود؛ دفعه‌های بعد معمولاً دوباره کد نمی‌خواهد.

### نوار وضعیت

پایین پنجره تعداد **پروکسی‌های سالم / کل** برای MTProto و V2Ray را نشان می‌دهد.

### لاگ (متن سیاه/سفید)

هر کاری که برنامه انجام می‌دهد آنجا نوشته می‌شود. اگر خطایی باشد، معمولاً همانجا می‌بینید.

## برنامه در پشت صحنه چه می‌کند؟

به زبان ساده:

1. **اول** سعی می‌کند با یک پروکسی MTProto به تلگرام وصل شود؛ اگر نشد، **مستقیم** (بدون پروکسی) وصل می‌شود.
2. **پیام‌های اخیر** کانال‌های `sources` را می‌خواند و لینک‌های `tg://proxy`، `t.me/proxy` و لینک‌های V2Ray را پیدا می‌کند.
3. لینک‌های **تکراری** را نادیده می‌گیرد؛ فقط **جدیدها** را در پایگاه داده می‌گذارد.
4. **آنلاین می‌ماند** تا پست جدید بیاید و همان لحظه لینک‌های تازه را بردارد.
5. هر **۳۰ دقیقه** (قابل تغییر در تنظیمات) دوباره پروکسی‌ها را امتحان می‌کند.

### پروکسی‌های Fake TLS (`ee…`)

بسیاری از پروکسی‌های جدید MTProto «secret» آن‌ها با **`ee`** شروع می‌شود. این برنامه برای همین نوع هم پشتیبانی دارد؛ برخلاف بعضی ابزارهای قدیمی که این لینک‌ها را نمی‌فهمند.

## فایل‌ها و پوشه‌ها — کجا چی ذخیره می‌شود؟

| مسیر | محتوا |
|------|--------|
| **`config.yaml`** | تنظیمات شما (API، کانال‌ها، …) — **این را backup بگیرید** |
| **`data/catalog.db`** | پایگاه دادهٔ همهٔ پروکسی‌ها (سالم و خراب) |
| **`data/subscription.txt`** | لیست پروکسی‌های V2Ray سالم — برای NekoRay |
| **`sessions/`** | فایل ورود تلگرام — **به اشتراک نگذارید** (مثل کلید حساب) |
| **`logs/`** | گزارش‌های ذخیره‌شده از پنل |
| **`xray/`** | برنامهٔ Xray برای تست V2Ray (اگر خودکار دانلود شده) |
| **`.venv/`** | محیط Python — لازم نیست دست بزنید |

## استفاده از اشتراک V2Ray در NekoRay

1. در پنل **Start Subscription server** را بزنید.
2. برنامه آدرسی شبیه این می‌دهد:

   `http://127.0.0.1:8765/subscription.txt`

3. در **NekoRay** (یا برنامهٔ مشابه):
   - گزینهٔ **Subscription / اشتراک** را باز کنید
   - این آدرس را به‌عنوان **URL اشتراک محلی** اضافه کنید
   - به‌روزرسانی کنید

فقط پروکسی‌های **تست‌شده و سالم** در فایل قرار می‌گیرند.

## تنظیمات پیشرفته (اختیاری)

در **`config.yaml`** می‌توانید این‌ها را هم عوض کنید:

| بخش | معنی ساده |
|-----|-----------|
| `scraper.proxy_check_interval` | هر چند ثانیه یک‌بار همه پروکسی‌ها دوباره امتحان شوند (پیش‌فرض ۱۸۰۰ = ۳۰ دقیقه) |
| `mtproto.max_working` | حداکثر چند پروکسی MTProto سالم نگه داشته شود (۰ = نامحدود) |
| `v2ray.max_working` | همین برای V2Ray |
| `v2ray.test_timeout` | چند ثانیه صبر کند تا بگوید پروکسی خراب است |

اگر نمی‌دانید چه بگذارید، **همان پیش‌فرض‌ها را نگه دارید**.

## مشکلات رایج و راه‌حل

### «setup.cmd خطا می‌دهد / Python پیدا نمی‌شود»

- اینترنت را چک کنید.
- یک‌بار **Command Prompt را به‌عنوان Administrator** باز کنید و دوباره `setup.cmd` را اجرا کنید.
- یا Python 3.10+ را از [python.org](https://www.python.org/downloads/) نصب کنید و هنگام نصب تیک **Add Python to PATH** را بزنید.

### «api_id / api_hash اشتباه است»

- دوباره از [my.telegram.org/apps](https://my.telegram.org/apps) کپی کنید.
- در `config.yaml` فاصلهٔ اضافه یا گیومک جاافتاده نباشد.

### «Scraper وصل نمی‌شود»

- `api_id` و `api_hash` درست باشند.
- کانال‌های `sources` را **درست** نوشته باشید و **عضو** آن‌ها باشید.
- اگر فیلترینگ شدید است، یک‌بار **Ping MTProto** بزنید یا موقتاً VPN دیگری روشن کنید تا **اولین ورود** انجام شود.

### «هیچ پروکسی پیدا نمی‌شود»

- کانال‌ها واقعاً لینک پروکسی می‌گذارند؟
- `messages_per_source` را بیشتر کنید (مثلاً ۱۰۰۰).
- Scraper را چند دقیقه روشن بگذارید.

### «Ping V2Ray کار نمی‌کند»

- مطمئن شوید `setup` کامل شده و پوشه **`xray/`** وجود دارد.
- بعضی لینکها فرمت‌هایی دارند که Xray پشتیبانی نمی‌کند؛ برنامه آن‌ها را «خراب» علامت می‌زند — طبیعی است.

### پنجره باز نمی‌شود

- از مسیر کامل استفاده کنید:

  ```bat
  D:\project\telegram-proxy-scraper\.venv\Scripts\pythonw.exe D:\project\telegram-proxy-scraper\app.py
  ```

- یک‌بار با `python.exe` به‌جای `pythonw.exe` امتحان کنید تا خطا در پنجرهٔ سیاه دیده شود.
